# Retrieval-Friendly Note Writing

This document teaches the agent how to write notes that are easy to find later.

Good retrieval depends heavily on note design.

## Title rules

Prefer titles that are:

- concept-shaped
- stable
- specific
- recognisable at a glance

Good titles:

- `active-outcome-beats-document-type`
- `avoid-catch-all-category-folders`
- `structural-changes-require-explicit-approval`

Bad titles:

- `notes-on-organisation`
- `thoughts`
- `misc`
- `2026-03-14`

## Summary rule

Put a short summary near the top of the note.

Example:

```md
# Avoid Catch-All Category Folders

> Use topic-, responsibility-, or outcome-based containers instead of generic buckets like `notes/`, `misc/`, or `working/`.
```

This summary should help a future agent recognise the note quickly.

## Linking rule

Add related links where useful.

Examples:

- parent topic index
- related atomic notes
- source topic note
- governing area note

This supports graph-style retrieval and recognition-based widening.

## Writing implication

When creating or refining durable notes, optimise not just for correctness, but also for future retrieval:

- title clarity
- summary quality
- alias usefulness
- topic labels
- related links
