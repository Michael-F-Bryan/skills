# Retrieval Anti-Patterns

Avoid these behaviours.

## Bad retrieval patterns

- treating the repository as raw text only
- starting with repo-wide grep
- ignoring PARA roots and container identities
- ignoring manifests and indexes
- deep-reading many notes before ranking candidates
- overusing archives for current questions
- assuming exact wording recall
- inventing matches from weak similarity
- creating new notes before checking whether equivalent knowledge already exists

## Bad note design patterns

- vague titles
- generic bucket containers
- missing summaries
- no aliases for commonly rephrased ideas
- no topic indexes for major domains
- poor linkage between related notes

## Failure mode

As the knowledge base grows, naive retrieval causes:

- rising token use
- lower precision
- duplicated knowledge
- more hallucinated matches
- slower workflows

This skill exists to prevent those problems.
