Current read

The handoff is intentionally simple. That is the point. The system should fail in obvious ways before it tries to recover in clever ways.

Recommendation

Use this artefact to explain the main flow, state boundaries, and operator responsibilities. Keep advanced recovery behaviour out of the first briefing.

System shape

The system has three useful boundaries: user intent, agent execution, and workspace state. Most confusion comes from mixing those boundaries together.

User intent → Agent plan → Tool execution → Workspace state → Handoff note
