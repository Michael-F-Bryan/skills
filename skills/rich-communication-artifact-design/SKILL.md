# Rich Communication Artefact Design

This is a framing overlay, not a second production workflow.

Express the job as:

> Reader needs to **do what** with **which information** so they can **take what action**?

Choose one dominant mode: understand, decide, compare, review, operate, or inspect evidence. If two modes compete, either subordinate one behind disclosure or split the artefact.

For ordinary HTML work, continue immediately with the ten-minute lane in `html-rich-communication`. Do not create a communication-contract file, content-disposition table, first-feedback document, or calibration ledger.

Ask at most one focused user question, and only when the missing answer would materially change the argument, recommendation, privacy posture, or scope. If a reasonable assumption preserves truth and safety, state it briefly and continue. Use a rough outline only when the unresolved answer produces genuinely different artefact shapes. Once the spine is settled, this skill is finished.
