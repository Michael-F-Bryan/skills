# Visual Elements Palette

Choose elements because they clarify the reader's task, not because they look impressive.

## Navigation and orientation

Use for long artefacts or mobile reading.

- Sticky section nav
- Question-based table of contents
- Progress rail / reading map
- Section summary strips
- Breadcrumb/current-context labels
- Back-to-top / next-decision links

## Progressive disclosure

Use when the surface must stay readable without discarding detail.

- Native `<details>/<summary>`
- Expandable evidence cards
- Layered depth: Plain English → More detail → Implementation notes
- Risk drawers
- FAQ accordions
- Collapsed code/diff snippets
- Caveat panels

## Structure and relationships

Use when the content is about how parts connect.

- Flow charts
- System architecture diagrams
- Tree diagrams
- Dependency graphs
- Swimlanes
- Layer-cake diagrams
- Hub-and-spoke maps
- Before/after architecture

## Process, timeline, and staged gates

Use when the reader needs order, investment stages, or decision points.

- Stage cards
- Decision gates
- Timeline strips
- Milestone ladders
- Roadmap lanes
- Experiment cards
- Kill/pivot panels

## Comparison and trade-offs

Use when the reader must choose between options.

- Option cards
- Side-by-side comparison grids
- Trade-off spectrums
- Decision matrices
- Pros/cons with verdict bands
- Small multiples
- Cost/effort/value quadrants

## Evidence, confidence, and uncertainty

Use when trust calibration matters.

- Confidence badges
- Evidence chips
- Assumption ledger
- Known/unknown panels
- Risk matrices
- Evidence gates
- Caveat callouts
- Source-backed quote blocks

## Data and quantitative displays

Use when numbers or patterns matter.

- Metric cards
- Sparklines
- Ordered bar charts
- Stacked bars for simple composition
- Heatmaps for density/risk/frequency
- Scatter/quadrant plots
- Annotated charts
- Conceptual maps when geography is the point

## Technical review aids

Use for architecture, PRs, incidents, and implementation plans.

- Annotated code snippets
- Diff views
- Module maps
- Call-flow diagrams
- State-machine diagrams
- Trace timelines
- Failure-path diagrams
- Contract cards

## Interaction and manipulation

Use only when the reader needs to explore, filter, tune, triage, or export.

- Filter chips
- Tabs/segmented controls
- Sort/group controls
- Sliders or scenario knobs
- Copy/export buttons
- Checklists
- Editable fields with live preview
