# Communication contract

The communication contract is the smallest useful description of what the artefact must do for its reader. Write it before implementation.

## Contract template

```markdown
## Communication contract

- Reader:
- Reader proximity: expert / adjacent / new / mixed
- Reader mode: decide / review / understand / operate / learn / compare / explore / edit-export
- Primary question the first screen must answer:
- Secondary questions:
- Action after reading:
- What must be believed/trusted:
- What must not be implied:
- Boundaries / non-goals likely to matter:
- Content that belongs elsewhere:
- Evidence available:
- Evidence missing:
- Privacy posture:
```

## Reader gap model

Use Google's audience equation:

```text
knowledge/skill the reader needs − reader's current knowledge/skill = explanation gap
```

Role is not enough. Record proximity to the domain, trust blockers, jargon risk, and what the reader will do next.

## Critique-first prompts

Before drafting, attack the contract:

- Does the primary question match the action after reading?
- What would this reader expect to see first?
- What likely non-goal or caveat will they wonder about early?
- Which source details belong in a different artefact?
- Which claim could be mistaken for stronger evidence than we have?
- What would a sceptical reviewer ask in the meeting?

If the contract fails, fix it before designing the page.
